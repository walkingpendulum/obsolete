Игра состоит из игровых дней. В течение каждого дня команды по очереди делают ход. Ход команды состоит из хода базы и хода всех имеющихся у команды муравьев.

База.

База должна быть наследником класса Base (расположен в файле Base.py), сигнатуры конструкторов должны совпадать. Но при этом в конструкторе можно дополнительно вводить любые свои поля.

Поле team_id — идентификатор, служит для того, чтобы различать команды.

Поле API — экземпляр программного интерфейса игры, с помощью методов которого ваши классы будут взаимодействовать с игровым миром и получать из него информацию о состоянии поля.

У класса должен быть реализован метод advance() — ход базы. Он не должен ничего возвращать, в нем ваша база может запрашивать состояние поля, обрабатывать его и строить планы по порабощению мира.

Муравей.

Муравей должен быть наследником класса Ant (расположен в файле Ant.py), сигнатуры конструкторов также должны совпадать. Аналогично, можно создавать любые свои поля.

Поле base — ссылка на свою базу. Через нее муравей может использовать API, а может просто забирать решение о собственном ходе, принятое на этапе хода базы.

У класса должен быть реализован метод move(). Метод должен возвращать кортеж (coord, string), где string — модификатор хода, одна из строк 'move', 'hit', 'drop_food', 'take_food', а coord — кортеж новых координат. Новые координаты должны отличаться от старых не более чем на единицу в каждой компоненте (проверки пока нет, но она будет).

Если модификатор равен 'move', то муравей будет перемещен в клетку с координатами coord, за исключением следующих случаев: если клетка находится за пределами игрового поля, если в клетке находится чья-либо база, если в клетке находится муравей (не важно, свой или чужой). Во всех перечисленных случаях ход этого муравья будет пропущен и муравей останется на месте. Также можно просто отдать старые координаты, тогда муравей останется на месте, это равносильно пропуску хода.

Если модификатор хода равен 'hit', то муравей попытается напасть на муравья в клетке с координатами coord, за исключением следующих случаев: если клетка находится за пределами поля, если в клетке нет муравья (а, например, чья-нибудь база). Во всех перечисленных случаях ход муравья будет пропущен. При нападении вероятность убить муравья (сейчас) равна 0.5. Если нападение завершится удачей, муравей, подвергшийся нападению, сразу будет убит (т.е. следующий ваш муравей ничего не найдет в этой клетке). В ином случае никаких последствий нападения не имеется. Обратите внимание, что проверки "свой-чужой" при нападении не производится, если вы нападете на своего муравья, то он тоже может быть убит.

Если модификатор хода равен 'take_food', то муравей попытается взять еду c клетки с координатами coord. Максимальное количество еды, которое может унести муравей, равно 1. Если он уже нес еду, то она исчезает. Если в клетке с указанными координатами нет еды, то муравей пропускает ход. Если еда есть, то он берет единицу еды.

Если модификатор хода равен 'drop_food', то муравей сбрасывает еду, если она у него была, иначе ничего не происходит. Если муравей сбрасывает еду на базу (свою или чужую), счетчик еды команды, которой принадлежит база, увеличивается на единицу. Если муравей сбрасывает еду на клетку, где уже лежит еда, счетчик количества еды в этой клетке увеличивается на единицу. Если муравей сбрасывает еду на другого муравья (своего или чужого), то происходит передача еды (нынешний муравей больше не имеет еды, новый муравей ее получает). Если муравей сбрасывает еду за пределы поля, то еда исчезает.

Формат файла с кодом.

Код должен находиться в одном .py-файле, который нужно положить в папку strategies. Внутри файла должны быть реализованы классы CustomBaseClass и CustomAntClass, а также написан следующий кусок кода:

MANIFEST = { 'description': '%Your strategy description here', 'BaseClass': %YourCustomBaseClass, 'AntClass': %YourCustomAntClass }

Обратите внимание, что два последних значения -- не строки, а объекты типа type. Примеры ищите в папке strategies

API

get_list_of_ants() — возвращает копию списка имеющихся у вас муравьев.

get_cost_of_ant_spawn() — возвращает стоимость создания нового муравья.

get_team_id_by_base(base) - возвращает идентификатор команды по экземпляру базы.

get_size_of_world() - возвращает кортеж (size_x, size_y) размеров игрового поля (индексация начинается с нуля).

is_enemy_by_coord(coord) - возвращает True или False, в зависимости от того, находится ли в заданной клетке муравей чужой команды или нет. Если в клетке нет муравья (а, например, база), или клетка находится за пределами поля, то возвращает False. В качестве координат принимает или два аргумента x, y, или кортеж (x, y).

get_coord_by_obj(obj) - возвращает кортеж координат заданного объекта: базы или муравья. Если объект не будет найден, вернет None.

get_type_by_coord(coords) -- возвращает тип объекта, хранящегося в клетке с координатами coords. Если клетка находится за пределами поля, то считается, что в ней хранится None. Возвращаемые методом типы: NoneType, Food (хранится в World.py), кастомный класс базы, кастомный класс муравья. Последние два можно распознавать при помощи issubclass(X, Base) или issubclass(Y, Ant). В качестве coords могут быть как два аргумента x и y, так и кортеж (x, y).

get_food_load(self, args) - возвращает количество еды. Если в качестве аргумента дан экземпляр базы, то метод вернет количество еды у команды, владеющей этой базой. Если в качестве аргумента дан муравей, то метод вернет количество еды (0 или 1), которую он имеет. Если в качестве аргумента заданы координаты -- пара чисел или кортеж -- то метод вернет количество еды, хранящейся в клетке с этими координатами. Если клетка за пределами карты либо в ней ничего нет, то метод вернет 0. Если в клетке будет муравей (свой или чужой), то метод вернет количество еды (0 или 1), которую он имеет. Если в клетке будет база (ваша или чужая), то метод вернет количество еды у команды владеющей этой базой.

ask_for_spawn([AntClass]) — попытка создания нового муравья. Имеет необязательный аргумент AntClass. Если аргумент указан, то муравей будет объектом указанного класса (на случай, если у вас имеется иерархия профессий, можно отдавать сюда разные подклассы), иначе будет создан муравей дефолтного класса. При создании случайным образом выбирается соседняя с базой клетка, после чего происходит проверка, свободна ли она и достаточно ли имеющейся еды. Если клетка оказалась занята (едой, другой базой или муравьем), то попытка создать муравья завершается неудачей. Если еды недостаточно, попытка создать муравья завершается неудачей. Иначе муравей будет создан и помещен в выбранную клетку, а необходимое количество еды будет списано со счета вашей команды. Метод возвращает True, если создание прошло успешно и False иначе.
